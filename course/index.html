首页如何刷新列表

把 getData 想成重新拍照：数据库里的摆设变了，你就再按一次快门，state 里的快照更新，列表就换新的。

插入后为什么要立刻 getData？

withTransactionAsync 保证 INSERT 和随后的读取在同一捆里完成。

CODE 
  async function insertTransaction(transaction: Transaction) {
    db.withTransactionAsync(async () => {
      await db.runAsync(
        `
        INSERT INTO Transactions (category_id, amount, date, description, type) VALUES (?, ?, ?, ?, ?);
      `,
        [
          transaction.category_id,
          transaction.amount,
          transaction.date,
          transaction.description,
          transaction.type,
        ]
      );
      await getData();
    });
  }
白话

定义异步函数：接收一笔完整的 Transaction 对象。

向 SQLite 要一个事务块，里面的步骤要么一起提交要么一起回滚。

执行一条写操作 runAsync。

多行字符串开始：里面是标准 INSERT 模板。

列名与占位符一一对应，顺序和下面数组一致。

模板字符串结束，逗号后是参数数组。

数组开始：按问号顺序塞值。

分类外键。

金额。

日期时间戳（秒）。

备注文本。

收入或支出枚举。

数组结束。

runAsync 调用结束。

写完后立刻再查一遍，刷新 transactions 等 state。

事务回调结束。

insertTransaction 函数体结束。

05

图表与周视图

柱状图像一周七格电量条：SUM 按星期几分组，strftime 负责从时间戳里读出「星期几」。

SQLite 里怎么按「星期几」加总？

CODE 
      const query = `
      SELECT 
        strftime('%w', date, 'unixepoch') AS day_of_week,
        SUM(amount) as total 
      FROM Transactions 
      WHERE date >= ? AND date <= ? AND type = ? 
      GROUP BY day_of_week
      ORDER BY day_of_week ASC
    `;
白话

在 TypeScript 里拼一段多行 SQL 字符串。

查询开始：选出两列。

把 date 列当 Unix 秒解释，格式化成星期索引 0–6（%w）。

同一天的多笔金额用 SUM 合成一个 total。

数据来源：Transactions 表。

只统计当前周起止时间戳之间、且类型为收入或支出的行。

按星期几分组，每天最多一条汇总。

从周日索引开始顺序输出，方便后面画柱子。

模板字符串结束，稍后用 getAllAsync 传三个参数绑定三个问号。

processWeeklyData 在补什么洞？

SQL 只返回「有交易的日子」；图表库却要固定七天长度。ChartQuery.ts 先把七天预填 0，再把查询结果塞回对应下标。

CODE 
  data.forEach((item) => {
    // Assuming item.dayOfWeek is in the range 0-6 (matching SQLite %w output)
    const dayIndex = item.dayOfWeek;
    if (dayIndex >= 0 && dayIndex < 7) {
      barData[dayIndex].value = item.total;
      if (item.total < 100) {
        barData[dayIndex].frontColor = "#d1d5db"; // gray for zero values
        barData[dayIndex].gradientColor = "#d1d5db"; // gray for zero values
      } else {
        barData[dayIndex].frontColor = isIncome ? "#d3ff00" : "#ffab00"; // default income/expense colors
        barData[dayIndex].gradientColor = isIncome ? "#12ff00" : "#ff0000"; // default income/expense gradients
      }
    } else {
      console.error(`星期索引无效: ${item.dayOfWeek}`);
    }
  });
白话

遍历 SQL 聚合后的每一天结果。

注释提醒：dayOfWeek 应对齐 SQLite 的 0–6。

取出当前项的星期下标。

若下标合法，进入更新逻辑。

把这一天的合计写进 barData 对应柱子的 value。

若总额低于 100，视为「视觉上的零花」，用灰色柱子。

灰色渐变，和前景一致。

否则用亮绿系表示收入、橙红系表示支出。

渐变第二色，让 gifted-charts 的 showGradient 有渐变可画。

if 分支结束。

内层 else：金额达到阈值，改用高对比收入/支出配色。

内层 if/else 整块结束。

下标异常时打错误日志，避免默默画错柱。

else 结束。

forEach 回调结束：七天数组现在填满。
